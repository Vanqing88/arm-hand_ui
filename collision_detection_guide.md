# 机器人碰撞检测功能使用说明

## 功能概述

本项目已成功集成了基于 Cannon.js 物理引擎的机器人碰撞检测功能，类似于 MoveIt 的碰撞检测效果。当机器人模型发生碰撞时，碰撞部分会高亮显示为蓝色，并且在拖拽滑块规划时也会实时显示碰撞状态。

## 主要功能特性

### 1. 实时碰撞检测
- 使用 Cannon.js 物理引擎进行精确的碰撞检测
- 支持机器人各个链接之间的自碰撞检测
- 实时更新碰撞状态，响应机器人姿态变化

### 2. 可视化碰撞反馈
- 碰撞部分高亮显示为蓝色（类似 MoveIt）
- 碰撞状态优先于关节极限警告显示
- 自动恢复原始材质当碰撞结束时

### 3. 状态面板显示
- 底部状态面板显示实时碰撞状态
- 显示碰撞数量和碰撞对信息
- 红色指示器表示检测到碰撞，绿色表示无碰撞

### 4. 拖拽滑块支持
- 在拖拽滑块规划路径时实时检测碰撞
- 计划值和实时值都能触发碰撞检测
- 支持所有控制模式（手臂控制、末端控制、预设动作）

## 技术实现

### 核心组件

1. **CollisionDetector.js** - 碰撞检测核心引擎
   - 基于 Cannon.js 物理引擎
   - 为每个机器人链接创建物理体
   - 实时更新物理体位置和姿态
   - 监听碰撞事件并触发回调

2. **CollisionStatus.js** - 碰撞状态显示组件
   - 显示碰撞状态指示器
   - 提供碰撞详情信息
   - 集成到底部状态面板

3. **RobotViewer.js** - 机器人可视化组件（已更新）
   - 集成碰撞检测系统
   - 处理碰撞可视化效果
   - 管理材质状态（原始、警告、危险、碰撞）

### 颜色编码系统

- **蓝色（#0080FF）**: 碰撞状态（最高优先级）
- **红色（#FF0000）**: 关节极限危险状态
- **橙色（#FF8C00）**: 关节极限警告状态
- **原始颜色**: 正常状态

## 使用方法

### 1. 启动应用
```bash
npm start
```

### 2. 操作机器人
- 使用滑块控制机器人关节
- 拖拽3D模型中的机器人部件
- 执行预设动作

### 3. 观察碰撞检测
- 当机器人部件发生碰撞时，碰撞部分会变为蓝色
- 底部状态面板会显示碰撞状态
- 碰撞结束后自动恢复原始颜色

## 性能优化

### 1. 碰撞过滤
- 相邻链接之间的碰撞被过滤，避免误报
- 特定链接（如 base_link）被排除在检测之外

### 2. 更新频率控制
- 物理世界以 60Hz 频率更新
- 视觉更新以 10Hz 频率进行，减少计算负担

### 3. 材质管理
- 使用材质缓存避免重复创建
- 智能材质状态管理，只在需要时更新

## 测试建议

### 1. 基本碰撞测试
1. 启动应用
2. 进入手臂控制模式
3. 拖拽左臂和右臂滑块，使两臂相互接近
4. 观察当两臂接触时是否变为蓝色
5. 查看底部状态面板是否显示碰撞状态

### 2. 自碰撞测试
1. 拖拽单个手臂的滑块
2. 使手臂弯曲到极限位置
3. 观察手臂是否与躯干或其他部位发生碰撞
4. 检查碰撞检测是否正确触发

### 3. 动态碰撞测试
1. 使用预设动作功能
2. 执行复杂的机器人动作
3. 观察动作过程中的碰撞检测
4. 验证碰撞状态的实时更新

## 故障排除

### 1. 碰撞检测不工作
- 检查浏览器控制台是否有错误信息
- 确认 Cannon.js 库已正确加载
- 检查物理体是否正确创建

### 2. 性能问题
- 减少物理世界更新频率
- 优化碰撞检测范围
- 检查是否有内存泄漏

### 3. 可视化问题
- 检查材质是否正确保存和恢复
- 确认碰撞状态回调是否正常工作
- 验证Three.js材质系统是否正常

## 未来改进方向

1. **更精确的碰撞检测** - 使用凸包或网格碰撞检测替代盒子碰撞检测
2. **碰撞距离检测** - 在实际接触前提供预警
3. **碰撞力反馈** - 计算碰撞力度并提供相应的视觉反馈
4. **碰撞避让** - 自动调整机器人姿态避免碰撞
5. **碰撞记录** - 记录碰撞历史和统计信息
