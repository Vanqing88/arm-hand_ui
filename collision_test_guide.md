## 测试步骤

### 1. 启动应用
```bash
npm start
```

### 2. 基本功能测试

#### A. 检查控制台
1. 打开浏览器开发者工具 (F12)
2. 查看控制台输出
3. 应该看到类似以下的日志：
   ```
   初始化碰撞检测系统...
   开始为机器人创建物理体...
   为链接 TORSO 创建物理体，包含X个网格，尺寸: X.XXX x X.XXX x X.XXX
   ...
   碰撞过滤已设置，相邻链接对: Set(34) {...}
   ```

#### B. 测试状态面板
1. 观察页面底部的状态面板
2. 初始状态应该显示：
   - 🟢 ROSBridge已连接（如果ROS正在运行）
   - 🟢 无碰撞
   - 🟢 系统正常

### 3. 碰撞检测测试

#### A. 手臂碰撞测试
1. 使用左右手臂的滑块控制
2. 拖动滑块使左右手臂相互接近
3. 观察现象：
   - 当两臂接触时，碰撞部分应该变为蓝色
   - 状态面板应该显示：🔴 检测到碰撞
   - 控制台应该输出：`有效碰撞: HAND_L <-> HAND_R`

#### B. 自碰撞测试
1. 拖动单个手臂滑块
2. 使手臂弯曲到与躯干接触
3. 观察现象：
   - 碰撞部分变为蓝色
   - 状态面板显示碰撞状态
   - 控制台显示有效碰撞信息

#### C. 相邻链接过滤测试
1. 查看控制台输出
2. 应该看到过滤信息：
   ```
   过滤相邻链接碰撞: SHOULDER_L <-> UPPERARM_L
   过滤相邻链接碰撞: TORSO <-> SHOULDER_R
   ...
   ```
3. 相邻链接不应该触发碰撞高亮

### 4. 动态测试

#### A. 实时规划测试
1. 拖动滑块时观察实时碰撞检测
2. 计划值改变时应该立即更新碰撞状态
3. 碰撞状态应该优先于关节极限警告

#### B. 预设动作测试
1. 切换到预设动作模式
2. 执行复杂动作
3. 观察动作过程中的碰撞检测

### 5. 性能测试

#### A. 检查帧率
1. 在浏览器开发者工具中查看性能
2. 应用应该保持流畅运行
3. 物理更新不应该显著影响性能

#### B. 内存使用
1. 检查内存使用情况
2. 长时间运行后不应该有内存泄漏
3. 物理体应该正确清理
